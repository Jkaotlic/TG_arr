### DEPLOY-03: Base image python:3.12-slim is an unpinned floating tag (no digest) — non-reproducible builds and silent CVE/behavior drift on rebuild
- **Файл**: `Dockerfile:1`
- **Проблема**: Both stages use `FROM python:3.12-slim` with no digest pin (`@sha256:...`) and no patch-level tag. Each rebuild on the rpie4 (e.g. after a code change or CI redeploy) pulls whatever `3.12-slim` currently points to, which Docker Hub re-publishes regularly with a new Debian point release and updated apt packages. This makes builds non-reproducible and means a regression or new CVE in the base can land without any code change. On ARM this also risks pulling a freshly-built arm64 image that broke something, with no way to roll back to the exact base that last worked.
